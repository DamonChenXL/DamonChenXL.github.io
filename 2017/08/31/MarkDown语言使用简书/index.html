MarkDown语言使用简书

第零节：写在前面

它是什么

  1. 可能对很多热衷于开源的朋友们来说，Markdown应该是一种非常熟悉的格式，但是可能对于很多朋友们来说，可能Markdown还是会感觉有点陌生
  2. 首先需要说明的是，Markdown是一种标记格式，我们通常用它来编写一些文本文件。。
  3. 该格式还可以被转换为其他格式，最经常被转换到的格式应该是HTML格式和PDF格式了，因为这两种格式也是应用非常的广泛。

建议评价

  1. 由于开源社区的大力追捧，因此我认为它是非常值得学习的。
  2. 当然很重要的一点就是，它并不复杂，是一个非常容易学习的内容。

第一节：光速入门

MarkDown 简述

  1. 在今天，MarkDown格式已经非常流行了，我们可以在很多开源项目里面看到后缀名带有”.md”或者”.markdown”或者”.mdown”格式的文件，它们就是我们介绍的Markdown格式的文件。

  2. 在程序员的圈子里，Markdown几乎是github和stackoverflow的御用语言，而且国内的很多博客、笔记、问答平台也在开始支持markdown。

  3. 它是一个非常轻量级的标记语言，创始人是John Gruber，该语言的目标是打造一个易写易读的标记语言。

  4. 我们可以把markdown格式转换成很多其他的格式，比如html、pdf、epub等等，比如html用于在网页上浏览，比如PDF用于打印和阅览。

  5. 而且Markdown接收了很多来自其他语言的优点，而HTML语言对它的影响则颇为显著，我们可以看到很多HTML的影子，当然这其中很重要的一点原因就是互联网的流行。

Markdown的使用

  1. 由于Markdown只是一种标记格式，而我们书写成这种格式的时候，通常还需要借助于一个工具来把它展示为更适合阅读的格式。

  2. 不过需要说明的是，不同的工具对Markdown语法的支持程度是不同的，有的Markdown编辑器对部语法支持的不好，而有些又新增加了一些自己定制的语法。

  3. 这些工具有在线的，也有离线的，我们今天就以离线的MdCharm为例，来介绍一下Markdown的语法。
